# Incremental Changes
- when planning a large feature, the changes should be broken down into incremental changes
- you SHOULD ask for user confirmation of work done before moving to the next step

# Notes
- planned features are often detailed in the `notes/` directory
- you SHOULD check there when planning a feature to look for guidance
- if a feature has multiple steps listed in the notes, indicate when each step is done
- sometimes work done affects other planned work - update related notes as work is complete
