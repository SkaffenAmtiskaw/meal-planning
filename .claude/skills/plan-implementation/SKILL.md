# Incremental Changes
- when planning a medium to large feature, the changes should be broken down into incremental changes
- you SHOULD ask for user confirmation of work done before moving to the next step
- you SHOULD NOT expand the scope past what the user requested - if scope is unclear prompt the user for clarification

# Notes
- planned features often have preliminary details in the `notes/` directory
- you SHOULD check there when planning a feature to look for guidance
- when creating a plan - you should add it to the note - indicate which part is generated by Claude and which parts are user generated
- if planned work has multiple steps and there is no existing note prompt the user and ask if one should be created
- while implementing the feature, update the note page to indicate what has been done
- sometimes work done affects other planned work - at the end of a feature YOU MUST check if this feature will update other planned and update their notes
