Colour palette

These are the five hex values to register in your Mantine theme. Use the names below as token identifiers throughout the codebase.

#1C3144 — navy
#44633F — forest
#B1BA95 — sage
#FF6542 — ember
#EFE7E9 — chalk

Mantine requires a full 10-stop colour ramp for any custom primary colour. Generate ramps for navy and ember at mantine.dev/colors-generator by pasting the hex. You only need to do this for the two colours you'll use as primaryColor candidates.

MantineProvider theme config

Wire this into your root MantineProvider. Comments indicate which decisions affect which screens.

const theme = createTheme({

  // Typography — swap Inter in via Google Fonts or next/font
  fontFamily: 'Inter, sans-serif',
  fontFamilyMonospace: 'JetBrains Mono, monospace',

  // Primary colour drives buttons, links, active states.
  // Use 'ember' for a warm energetic feel, or 'navy' for subdued professional.
  // Recommend: ember as primaryColor, navy as the shell colour (navbar/sidebar).
  primaryColor: 'ember',
  primaryShade: { light: 5, dark: 6 },

  // Consistent rounding everywhere — md is a good default for a utility app
  defaultRadius: 'md',

  // Register your custom colour ramps (generated from mantine.dev/colors-generator)
  colors: {
    ember: [/* 10 stops from generator */],
    navy:  [/* 10 stops from generator */],
  },

  // Shell/layout colours
  // The AppShell header and navbar should use navy as their bg
  components: {
    AppShell: {
      styles: {
        header: { backgroundColor: '#1C3144' },
        navbar: { backgroundColor: '#1C3144', borderRight: 'none' },
      },
    },

    // Nav links in the sidebar — active state should use sage/ember, not the blue default
    NavLink: {
      styles: (theme) => ({
        root: {
          color: '#EFE7E9',               // chalk — readable on navy
          '&[data-active]': {
            backgroundColor: '#44633F',   // forest green active state
            color: '#EFE7E9',
          },
          '&:hover': {
            backgroundColor: 'rgba(255,255,255,0.08)',
          },
        },
        label: { fontWeight: 500 },
      }),
    },

    // Buttons — ember primary, consistent sizing
    Button: {
      defaultProps: { radius: 'md' },
    },

    // Inputs — slightly warmer border to match chalk palette
    Input: {
      styles: {
        input: { backgroundColor: '#FFFFFF', borderColor: '#C8C0C3' },
      },
    },

    // Tabs — used on the Settings page
    Tabs: {
      styles: (theme) => ({
        tab: {
          '&[data-active]': {
            borderColor: '#FF6542',   // ember underline
            color: '#1C3144',
          },
        },
      }),
    },

    // Modal — used by Add Meal
    Modal: {
      defaultProps: { radius: 'md', overlayProps: { blur: 2 } },
    },
  },
});

Calendar view (schedule-x)

Schedule-x uses its own CSS custom properties, separate from Mantine. You'll need to set these on the calendar's wrapper element. It also supports custom event components.

/* Target the schedule-x root — wrap your calendar in a div with class="sx-cal-wrapper" */
.sx-cal-wrapper {

  /* Primary brand colour — used for today highlight, selected states */
  --sx-color-primary: #FF6542;
  --sx-color-primary-container: rgba(255, 101, 66, 0.12);

  /* Surface colours */
  --sx-color-surface: #FFFFFF;
  --sx-color-surface-variant: #EFE7E9;  /* chalk for alternating rows etc */

  /* Text */
  --sx-color-on-surface: #1C3144;
  --sx-color-on-surface-variant: #5A6A7A;

  /* Grid lines */
  --sx-color-outline: #DDD8DA;

  /* Today cell */
  --sx-color-today-bg: rgba(255, 101, 66, 0.08);
}
Check the schedule-x docs for the exact CSS variable names — they may differ slightly by version. The pattern above follows their v1 naming. Inspect the rendered calendar in devtools to confirm the variable names in use.

For the "today" date badge (currently purple), override it to ember. If schedule-x renders it as a hardcoded colour rather than a CSS var, you may need a targeted selector like .sx__month-grid-day__header-date--today.

The toolbar above the calendar (Today button, Month/Week/List segmented control, Date picker) is your own Mantine components, so those just inherit the theme. Make sure the segmented control uses color="ember".

Font recommendation

Inter is a safe, neutral choice that pairs well with this palette. For something with slightly more character that still reads well in a data-heavy UI, consider DM Sans — it has a friendlier feel that suits a meal planner without being informal.

Add via Google Fonts in your HTML head:

<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600&display=swap" rel="stylesheet">

Then set fontFamily: 'Inter, sans-serif' in the theme as shown above.
